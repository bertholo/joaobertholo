Introduction

Budget Buddy is a full-stack application that I am currently working on. It is originally an expense tracker with a transactions history section and some graphs displaying the balance, current month espenditure and history chart. New features will be added in the future.

Built using React, Bootstrap and Chart.js as Front End technologies, as well as Node.js with the Express.js library and MongoDB as the database.

On the Front End, the App manages State globally with useContext that handles the API requests for incomes and expenses via the useState hook and the errors as well. On the App.js file, State is managed by a function that uses a switch statement to determine which Component to display. The Default is set to be the Dashboard Component.

The fun side of this project was to be working with charts via Chart.js. It is so many types of charts, datasets and options that you can spend some time playing with all the different layouts. Even though it was fun to work with, it was challenging trying to connect the data from the input of incomes or expenses and the dates with the datasets from the charts.

The charts counts with moment.js to set the date format and some helper functions to get the months from transactions, amounts, sort the months in order and group the amounts toguether depending on the chart. If there is no input for a certain month, its amount is set to zero.

Incomes and Expenses pages are very similar in layout and functionality. Both of them have an unique icon for each income or expense, there is an input form that adds a new item to the database with the app updating immediately, as well as a delete button that deletes the item from the database with the instant update of the App and finally the total income or expense displayed at the top.

On the Back End side, I use morgan for HTTP request logging and cors to manage cross-origin resource sharing. The Application use the standard RESTful API to improve scalability and flexibility.

The database has two models: Income and Expense. Each model has a title, amount, type, date, description, and category. Validation acordingly. The Schemas were created with mongoose.
