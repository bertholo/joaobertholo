Project RAVENOUS

front-end Project

Ravenous is my first React application built on HTML5, JavaScript and CSS3. It is essentially a restaurant recomendation dynamic website that allows users to search for local businesses based on their location, term and sorting options. All of this made possible with the implementation of asyncronous requests for the yelp API.

Ravenous is simple and dynamic with the implementation of components that manages state using 'useState' hook, dynamic styles for scalability and props for making the project reusable and modular.

Here is how it works:

There is a component named SearchBar responsible for aking the get request for the Yelp API, you enter the location of your choice and the term(eg: japanese food). The SearchBar component has an initial State of "Best Match", so it defaults to it if no sorting option is selected. There is an error handling Try-catch block for handling errors during the request. If the request is successful, an status of 200 will be logged to the console and an array of restaurants data will be sent by the API as a json response. The otherwise is that a message error will be logged along with the status response.

Then the BusinessList component is responsible for mapping through the response creating instances of each business. The relevant data is stored in each Business component and the result is an array of mock restaurants displayed to the user.

The project does not work with a server or database and the businesses dislpayed are not selectable. Functionality will be developed soon.

Note that if the project is to be forked (which involves creating your own copy of this repository on GitHub) from GitHub, an yelp api key has to be provided to the code.

The project is using cors-anywhere, which serves as a proxy server and adds essential CORS headers for development purposes.
